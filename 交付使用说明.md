# 销售支持 Agent｜交付使用说明 v1

## 1. 项目说明

这是一个面向销售支持场景的本地化 Agent 原型系统，当前核心能力包括：

- 客户分析
- 资料检索
- 话术生成

系统默认基于本地规则、本地知识数据和本地模型运行，优先满足“数据尽量本地处理”的要求。

---

## 2. 启动方式

进入项目根目录后，执行：

```bash
./runtime/start.sh
```

正常启动后，可通过以下地址访问：

- 后端健康检查：`http://localhost:3001/health`
- Web 前端：`http://localhost:5173/`

---

## 3. 配置文件

项目运行依赖以下配置文件：

- `config/model.env`
- `config/database.env`

首次使用时，可参考以下模板文件：

- `config/model.env.example`
- `config/database.env.example`

### 3.1 模型配置

当前默认模式为本地模型模式，主要字段包括：

- `MODEL_PROVIDER`
- `MODEL_MODE`
- `HOST`
- `PORT`
- `LOCAL_LLM_BASE_URL`
- `LOCAL_LLM_MODEL`
- `LOCAL_LLM_TIMEOUT_MS`

### 3.2 数据库配置

当前默认数据库模式为 SQLite，主要字段包括：

- `DB_TYPE`
- `SQLITE_PATH`

如果后续切换 PostgreSQL，可填写：

- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`

---

## 4. 测试方式

当前建议优先测试 Web 版主链路：

1. 首页
2. 客户分析
3. 资料检索
4. 话术生成

重点验证内容：

- 页面是否正常打开
- 是否能正常返回结果
- 本地模型链路是否正常
- `/health` 是否正常返回

### 4.1 健康检查

执行：

```bash
curl http://localhost:3001/health
```

若返回：

```json
{
  "success": true,
  "message": "service is healthy",
  "data": {
    "service": "sales-support-agent",
    "status": "ok"
  }
}
```

则说明服务运行正常。

---

## 5. 数据文件维护

当前核心数据文件包括：

- `data/rules.json`
- `data/products.json`
- `data/faqs.json`
- `data/script_templates.json`

它们分别用于：

- 规则匹配
- 产品定义
- FAQ / 风险提示
- 话术模板生成

后续如需新增产品、规则或 FAQ，应优先修改以上数据文件，而不是直接改页面逻辑。

---

## 6. 当前版本定位

当前版本为内部测试版，重点目标是：

- 验证 Web 主链路可用性
- 验证本地模型链路可用性
- 验证规则层、资料层、话术生成层联动正常

当前不以“小程序交付”为主线，主线是先把 Web 版收成可测试、可运行、可继续封装的版本。
